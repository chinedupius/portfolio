Objective

The purpose of this lab exercise was to configure Windows Defender Firewall rules to allow ICMP (ping) traffic for troubleshooting network connectivity between devices. Instead of disabling the firewall (which would increase risk), I created a temporary inbound rule to permit ICMP. After successful testing, I removed the rule to restore the secure baseline.

üîß Tools & Environment

  • Operating System: Windows 11
  • Firewall Utility: Windows Defender Firewall with Advanced Security
  • Command Line: Windows Command Prompt (ping tests)

üìå Procedure

  1. Verify Ping Test Before Rule
  2. Access Firewall Settings
  3. Open Inbound Rules
  4. Create a New Rule
  5. Define Protocol and Ports
  6. Name the Rule
  7. Set Action and Profile
  8. Verify with Ping Test
  9. Remove Rule After Testing
  10. Verify Ping Test After Troubleshooting

Results

  • The ICMP rule allowed successful pings between devices.
  • Once deleted, the firewall reverted to its secure default state.
  • Good practice demonstrated: enable targeted rules temporarily rather than disabling the firewall.

Conclusion

This lab reinforced secure firewall management. By adding a temporary inbound rule for ICMP, I troubleshot connectivity safely and then removed the rule to maintain least privilege on the host.
