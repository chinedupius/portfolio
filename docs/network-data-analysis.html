üî• Configuring Firewall Settings and Adding New Rules

This lab exercise focuses on configuring firewall rules to allow network troubleshooting by enabling ICMP (ping) requests. Rather than disabling the firewall entirely, which is considered a poor practice in production environments as it exposes the system to potential attacks, the recommended approach is to add a temporary firewall rule that permits ICMP traffic. This allows devices to be pinged for troubleshooting purposes while maintaining overall system protection. Once troubleshooting is complete, the temporary rule should be removed to restore the firewall to its original secure state. This method ensures both effective troubleshooting and continued protection of the system against unauthorized access.
